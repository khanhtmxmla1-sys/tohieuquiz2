# /flow

1. Tải bộ kỹ năng `@workflow-patterns`.
2. Kiểm tra trạng thái hiện tại của task và các thay đổi chưa commit.
3. Hướng dẫn thực hiện commit theo chuẩn Conventional Commits (feat, fix, refactor...).
4. Đảm bảo mọi thay đổi đều có logic rõ ràng và có thể hoàn tác (revert) an toàn.
5. Cập nhật `task.md` hoặc `plan.md` của dự án để phản ánh đúng tiến độ.
