## 🟢 1. Bảo mật: Secret Leakage (Rò rỉ thông tin)
- **Trạng thái:** ✅ An toàn.
- **Chi tiết:** File `.env` và các file cấu hình nhạy cảm đã được đưa vào `.gitignore`. Không phát hiện API Key hay Secret Token nào bị hardcode trực tiếp trong mã nguồn đẩy lên Git.
- **Hành động:** Tiếp tục duy trì việc sử dụng biến môi trường (Environment Variables) trên Vercel và Cloudflare Workers.

## 🟢 2. Bảo mật: XSS & SQL Injection
- **Trạng thái:** ✅ Tốt.
- **Chi tiết:** 
  - Các truy vấn database trong Workers đã sử dụng **Prepared Statements** (`bind()`), phòng chống hiệu quả tấn công SQL Injection.
  - Các đoạn hiển thị nội dung HTML (`dangerouslySetInnerHTML`) đã được đi qua hàm lọc `formatHtmlText` với danh sách trắng (whitelist) các thẻ an toàn.
- **Hành động:** Không cần thay đổi.

## 🟡 3. Bảo mật: Password Hashing (Mật khẩu học sinh)
- **Trạng thái:** ⚠️ Cần lưu ý.
- **Vị trí:** `workers/src/index.ts`
- **Chi tiết:** Hiện tại hệ thống sử dụng thuật toán **SHA-256** đơn thuần để lưu mật khẩu học sinh.
- **Nguy cơ:** Nếu database bị lộ, hacker có thể dùng kỹ thuật "Rainbow table" để dò ngược mật khẩu dễ dàng vì không có "Salt" (chuỗi muối ngẫu nhiên).
- **Cách sửa:** Nên kết hợp thêm một chuỗi "Salt" trước khi hash, hoặc chuyển sang dùng thư viện chuyên dụng nếu môi trường cho phép. Tuy nhiên, với quy mô trường học, điều này có thể tạm chấp nhận nếu so với việc lưu text thô.

## 🟢 4. Chất lượng Code: Dead Code & Imports
