## 🔴 NGHIÊM TRỌNG — Phải sửa ngay

### 1. Khóa API "giả" nằm chình ình trong code Frontend
**Vị trí:**
- `src/services/imageGenerationService.ts` → fallback `'123456'`
- `src/services/geminiService.ts` → fallback `'sk-dummy-key'` (2 chỗ)
- `src/domains/ai/ai.factory.ts` → fallback `'sk-dummy-key'`

**Hậu quả nếu không sửa:** Khi biến môi trường chưa được cấu hình (ví dụ lúc deploy nhầm), hệ thống sẽ gửi request với key giả → Dễ bị phát hiện và bị kẻ xấu lợi dụng lỗ hổng.

**Phác đồ:** Bỏ tất cả fallback key giả. Nếu thiếu key → throw Error rõ ràng thay vì chạy ngầm.

### 2. Thư viện Wrangler có lỗ hổng bảo mật HIGH
**Vị trí:** `workers/package.json` → `wrangler@3.80.0`

**Chi tiết:**
| Package | Mức độ | Mô tả |
|---------|--------|-------|
| `wrangler` | 🔴 HIGH | OS Command Injection trong `wrangler pages deploy` |
| `esbuild` | 🟡 Moderate | Dev server bị đọc response trái phép |
| `undici` | 🟡 Moderate | Unbounded decompression → nghẽn tài nguyên |
| `miniflare` | 🟡 Moderate | Kế thừa lỗi từ undici |

**Phác đồ:** Chạy `npm install --save-dev wrangler@4` trong thư mục `workers/`.

## 🟡 NÊN SỬA — Cải thiện chất lượng

### 3. 17 file Frontend còn `console.log`
Khi deploy production, các dòng `console.log` này vẫn hiện trong trình duyệt của học sinh → Lộ thông tin debug, giảm hiệu năng.

**Phác đồ:** Xóa hoặc thay bằng một logger có thể tắt/mở theo môi trường.

---

### 4. 11 chỗ dùng `dangerouslySetInnerHTML`
**Vị trí:** `PdfTab.tsx`, `IoeStudentView.tsx`, `QuestionRenderer.tsx`, `StudentDetailModal.tsx`

**Đánh giá:** Đang được bảo vệ qua hàm `formatHtmlText()` → **Mức độ nguy hiểm TRUNG BÌNH**. Tuy nhiên, nếu hàm sanitize bị bỏ sót 1 case đặc biệt, hacker có thể chèn mã độc (XSS).

**Phác đồ:** Cân nhắc dùng thêm thư viện `DOMPurify` để bảo vệ tuyệt đối.

---

### 5. 50+ lần dùng `any` type trong Backend Workers
**Vị trí:** `workers/src/utils/helpers.ts`, `workers/src/routes/*.ts`

**Hậu quả:** TypeScript mất khả năng kiểm tra lỗi → Bug "im lặng" có thể lọt qua mà không ai biết.

**Phác đồ:** Dần dần thay `any` bằng interface cụ thể. Ưu tiên các hàm xử lý data quan trọng (`mapQuestionForSave`, `handleValidateAnswers`).

---

### 6. 2 file backup cũ còn sót trong `src/`
- `src/components/student/ResultScreen.old.tsx`
- `src/components/student/ResultScreen.backup.tsx`

**Phác đồ:** Xóa 2 file này. Code cũ luôn có trong Git history nếu cần xem lại.

---

### 7. Database ID lộ trong `wrangler.toml`
`database_id = "c9482e3c-ff73-4f14-84af-52e7b6a580f4"` nằm trong file commit lên Git.

**Đánh giá:** Mức nguy hiểm THẤP (cần thêm API token mới truy cập được), nhưng không nên để lộ thông tin hạ tầng.

**Phác đồ:** Chuyển `database_id` sang biến môi trường hoặc chấp nhận rủi ro thấp (Cloudflare khuyến nghị để trong file).

## 🟢 GỢI Ý — Nâng cấp khi rảnh

### 8. Nâng cấp Wrangler lên v4
Version hiện tại `3.80.0` đã cũ. Version mới nhất `4.72.0` có nhiều cải tiến hiệu năng và bảo mật.

### 9. Thêm Rate Limiting cho API đăng nhập
API `/api/teachers/login` và `/api/students/login` chưa có giới hạn số lần thử → Kẻ xấu có thể brute-force mật khẩu.

### 10. Tối ưu Bundle Size
File JS lớn nhất sau build: `QuizPreview` (479 KB), `BarChart` (344 KB). Cân nhắc lazy-load các component nặng.

### 11. Thêm Error Boundary cho các component chính
Hiện có `ErrorBoundary.tsx` nhưng cần kiểm tra xem đã được wrap đúng chỗ chưa.
